`codegraph init` auto-adds `.codegraph/` to the project `.gitignore` (SPEC-001 branch, alongside the [[embedding-pass]] work — the local index now includes embedding vectors, tens of MB).

- Helper `ensureCodegraphIgnored(projectRoot): boolean` lives in `src/utils.ts` (among the fs helpers, before `FileLock`). Advisory: whole body in try/catch → returns false on ANY fs error, never throws (matches the repo "advisory" idiom used by the embedding pass).
- Git detection = `fs.existsSync(<root>/.git)` — true for BOTH a `.git` dir and a `.git` FILE (worktrees/submodules use a gitdir-pointer file). No git binary.
- Idempotent: skips if any line exactly equals one of `.codegraph`, `.codegraph/`, `/.codegraph`, `/.codegraph/` (trailing whitespace tolerated, leading NOT).
- Append is pure (never rewrites existing bytes): `separator = existing.endsWith('\n') ? '' : '\n'`, then a comment line + `.codegraph/`.

**Why:** the index (SQLite + vectors) is a machine-generated local artifact, fully regenerated by init/sync — must never be committed.

**How to apply:** wired ONLY in the CLI init action (`src/bin/codegraph.ts`, after the "Initialized in" success, prints via the standalone `info()` helper). Deliberately NOT called from `src/index.ts` — library/test/MCP init paths must stay byte-identical. If you touch init and wonder why the library doesn't gitignore, that's intentional, not an oversight. Tests: `__tests__/gitignore-init.test.ts`.
