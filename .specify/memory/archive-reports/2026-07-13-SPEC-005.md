SPEC-005 shipped `codegraph serve --web`: a zero-dependency `node:http` local
REST API riding the existing daemon/query-pool. **Slice 1 (#41)** delivered the
read surface — server + index health (`/api/status`), the machine list
(`/api/repos`), symbol search (`/api/search`), a symbol's detail with
callers/callees pages (`/api/nodes/:id`), impact radius (`/api/impact/:id`), and
graph neighborhood (`/api/graph`) — plus the security model: loopback-default
bind (`127.0.0.1:11235`), fail-closed on non-loopback unless
`CODEGRAPH_SERVER_TOKEN` is set (then required as a `Bearer` token on every
request), a Host-header allowlist, a closed six-code JSON error vocabulary, and
token-redacting request logging (FR-014a — the token never appears in a log in
any reversible form). **Slice 2 (#42)** added the job subsystem: `POST
/api/reindex/:repo` starts a background incremental sync or full rebuild over the
indexing entrypoints, `GET /api/reindex/:repo/events` streams live SSE progress,
one active job per project (409 on duplicate), a triggered job survives client
disconnect, and the endpoints carry the same Bearer auth on a network bind.
Dormant by default — without `--web` nothing binds and `codegraph serve` behaves
exactly as before. A static-asset mount point is reserved for SPEC-006 and a
WebSocket upgrade hook for SPEC-009.
