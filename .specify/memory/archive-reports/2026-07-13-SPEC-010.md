SPEC-010 shipped graph-aware symbol rename, delivered as two vertical slices.
**Slice 1 (#43, read-only)** delivered the plan engine and `codegraph rename
<target> <new-name>` in dry-run mode: name-based targeting with
ambiguity/kind-coverage refusals that teach precise targeting, an edit plan
listing every file touched with a before/after preview and a confidence rating
per edit, LSP-powered where a language server is available and graph-derived
everywhere else, `--json` for a machine-readable plan, and — critically — zero
writes to disk. **Slice 2 (#44, write)** added the apply path: a span guard that
re-verifies each edit location before touching bytes, atomic write with snapshot
rollback on failure, a post-check, and a targeted re-sync of the changed files;
the `codegraph_rename` MCP tool bringing the same contract into an agent's normal
loop; and the agent-facing tool-guidance update — added without regressing
existing retrieval. tasks.md closed 53/53.
