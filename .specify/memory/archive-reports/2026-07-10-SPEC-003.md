SPEC-003 shipped query-time hybrid semantic search: rank-only Reciprocal-Rank
Fusion (k=60) of the FTS5 keyword arm with a brute-force cosine vector arm over
the SPEC-001/002 embeddings; a `mode` parameter (`keyword|semantic|hybrid|auto`)
on the library (`searchNodes`/`searchNodesDetailed`, default `keyword`), MCP
`codegraph_search` (default `auto`), and CLI `codegraph query --mode` (default
`auto`); provenance tags (`matchType`/`fusedScore`, `[keyword]/[semantic]/[both]`)
and an embed/fusion timing footer; four literal success-shaped degradation hints
(no provider / no matching-model vectors / warming / embed-failure) that never
return `isError`; a `(count, model, dims, writeVersion)` staleness token whose
monotonic `vectors_write_version` counter invalidates the resident matrix on
same-count re-embeds and renames; a 1 GiB matrix memory guard; and a
`Hybrid search available` status line + additive `--json` fields. Dormant by
default — with no embedding env, keyword output is byte-identical to the
pre-feature baseline (no new env vars; activation rides the SPEC-002
`CODEGRAPH_EMBEDDING_*` config).
